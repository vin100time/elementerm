# SEO Session Rules

## Context
This is an SEO-focused Elementerm session. Follow these rules strictly.

## Technical SEO
- Ensure proper meta tags (title, description, canonical)
- Implement structured data (JSON-LD) where applicable
- Check robots.txt and sitemap.xml consistency
- Verify proper heading hierarchy (h1 > h2 > h3)
- Ensure all pages are crawlable and indexable

## Performance
- Target Core Web Vitals thresholds (LCP < 2.5s, FID < 100ms, CLS < 0.1)
- Optimize images (WebP, lazy loading, proper dimensions)
- Minimize render-blocking resources
- Implement proper caching headers

## Content
- Ensure unique, descriptive title tags (< 60 chars)
- Write compelling meta descriptions (< 160 chars)
- Use proper internal linking structure
- Avoid duplicate content across pages
- Implement hreflang for multilingual content

## Schema Markup
- Use JSON-LD format for structured data
- Validate with Schema.org specifications
- Cover relevant types: Organization, Product, Article, FAQ, etc.

## Git
- Conventional commits: feat:, fix:, chore:
- Document SEO changes in commit messages (which pages affected)
